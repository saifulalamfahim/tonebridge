# Support

Use [GitHub Issues](https://github.com/saifulalamfahim/tonebridge/issues) for public, reproducible bugs and feature proposals. Search existing issues first and use the provided templates.

Include the ToneBridge version, browser and operating-system versions, public page URL or minimal reproduction, and steps using invented text. Remove names, messages, credentials, tokens, and account details from screenshots and logs.

Security vulnerabilities do not belong in public issues. Follow [SECURITY.md](SECURITY.md) for private reporting.

ToneBridge is community-maintained open-source software. Support is best effort; there is no guaranteed response time, provider uptime, translation outcome, or compatibility with an undocumented editor.
